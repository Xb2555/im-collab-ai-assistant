    current-artifact-reference-score: 70
    continuation-keyword-score: 20
    continuation-audience-score: 25
    continuation-intent-cap: 100
    artifact-edit-anchor-score: 90
    artifact-edit-mutation-score: 35

## 先看哪些日志
真实 IM 黑盒先重点看这条日志：

```text
current_task_arbiter_decision
```

关键字段：

- `decision`
- `freshTaskScore`
- `currentTaskReferenceScore`
- `continuationIntentScore`
- `artifactEditScore`
- `newDeliverableScore`
- `ambiguousMaterialOrganizationScore`
- `topRecommendationId`
- `topRecommendationScore`
- `secondRecommendationId`
- `secondRecommendationScore`

辅助看：

- `pending_followup_preview_hint`
- `pending_followup_execution_hint`
- `completed_artifact_new_task_recovered`
- `completed_artifact_new_task_recovery_rejected`

## 怎么判断该调哪类参数
### 1. 明显新任务却被接到当前任务
常见现象：

- `decision=PROCEED_CURRENT_TASK`
- `newDeliverableScore` 不低，但 `topRecommendationScore` 很高

优先调：

- 降低 `affinity-source-*`
- 降低 `affinity-current-task-*`
- 提高 `proceed-current-task-min-score`
- 提高 `semantic-unique-gap`

如果用户句子里只是提到 `PPT/文档/摘要` 就被误承接，先降：

- `new-deliverable-mention-only-score`
- `affinity-deliverable-medium-score`

### 2. 明显 follow-up 却被放成新任务
常见现象：

- `decision=PROCEED_NEW_TASK`
- `topRecommendationScore` 在 60-74 之间
- `currentTaskReferenceScore` 或 `continuationIntentScore` 已经不低

优先调：

- 降低 `proceed-current-task-min-score`
- 提高 `current-task-reference-strong-score`
- 提高 `continuation-keyword-score`
- 提高 `affinity-source-*`

如果是单推荐场景老是差一点没过线，先调：

- `proceed-current-task-min-score`

### 3. 模糊 continuation 没走澄清，直接掉新任务
常见现象：

- `decision=PROCEED_NEW_TASK`
- `ambiguousMaterialOrganizationScore` 或 `continuationIntentScore` 已有明显值

优先调：

- 降低 `continue-current-task-min-score`
- 提高 `ambiguous-material-base-score`
- 提高 `continuation-keyword-score`

重点关注句式：

- `整理一下材料`
- `帮我梳理一下`
- `再整理一版`

### 4. 产物修改被误判成新任务或 follow-up
常见现象：

- 用户说 `改PPT，第二页...`
- 但 `decision` 不是 edit 相关绕过

优先调：

- 提高 `artifact-edit-anchor-score`
- 提高 `artifact-edit-mutation-score`
- 降低 `new-deliverable-mention-only-score`
- 提高 `artifact-edit-preference-min-score`

如果“改 PPT”因为提到 `PPT` 被误认为新交付物，优先调：

- `new-deliverable-mention-with-edit-anchor-score`

### 5. 多推荐本该唯一命中，却老是 ASK_SELECTION
常见现象：

- `topRecommendationScore` 比 `secondRecommendationScore` 只高一点点
- `decision=ASK_CURRENT_TASK_SELECTION`

优先调：

- 降低 `semantic-unique-gap`
- 提高 `affinity-deliverable-high-score`
- 提高 `affinity-source-high-score`

如果多推荐里有一个明显是 `PPT`、一个明显是 `SUMMARY`，但都打到高分，通常要收紧：

- `mentionsSummary` 相关权重
- `affinity-source-high-score`

## 推荐调参顺序
每次只改一组，避免互相干扰：

1. 先调阈值：
   `proceed-current-task-min-score`
   `proceed-new-task-min-score`
   `continue-current-task-min-score`
   `artifact-edit-preference-min-score`
2. 再调 recommendation affinity：
   `affinity-deliverable-*`
   `affinity-source-*`
   `affinity-current-task-*`
   `affinity-continuation-*`
3. 最后再调 evidence 原始分：
   `current-task-reference-*`
   `continuation-*`
   `artifact-edit-*`
   `ambiguous-material-*`

## 一轮真实 IM 建议观察样本
每次改完参数，至少回放这 8 类：

- `生成一份北京旅游ppt`
- `根据这个文档做一个汇报ppt，要4页`
- `帮我生成摘要`
- `根据这个任务内容生成一下汇报总结，我要发给老板`
- `整理一下材料`
- `帮我整理一份材料`
- `改PPT，第二页多加一点历史文化`
- `文档加一小节关于ggbond的内容`

## 判定是否调好了
可以先用这个简单标准：

- 明显新任务：直接 `PROCEED_NEW_TASK`
- 明显 follow-up：直接 `PROCEED_CURRENT_TASK`
- 模糊 continuation：优先 `ASK_NEW_OR_CURRENT`
- 明确产物修改：不进 follow-up，不进新任务
- 多推荐且不唯一：`ASK_CURRENT_TASK_SELECTION`

如果连续 20-30 条真实 IM 样本里，只剩个别边界句式需要调参，说明这套 routing tuning 已经进入可运营状态。
