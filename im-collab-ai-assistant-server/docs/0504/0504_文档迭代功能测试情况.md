## /api/planner/tasks/document-iteration 与/api/planner/tasks/{taskId}/document-iteration/approval

## 前言

锚点定位十分有问题；你必须查看lark_cli最新版本1.0.23的用法，按照架构设计文档，结合llm精准定位，而不是仅仅靠if和正则表达式硬编码来定位

下方列举刚刚遇见的几个问题（不清楚是否已经修复）

1. 定位不到，有时候输入`在1.3 基础设施与服务能力中`插入文字定位失败，而插入图片定位成功（重试了多次还是一样的结果）
2. 定位到后，插入/更新文字时，会把锚点标题变成普通正文，导致后端报错
3. 锚点对于定位文章开头无能为力，只能定位到大标题`一、`或者`1.1`，可能是代码里采用了正则表达式硬编码的原因

顺带提一下，创建文档写入的时候，疑似把评审节点的评审意见写入了文章开头，导致文章开头多了一段审核文字
