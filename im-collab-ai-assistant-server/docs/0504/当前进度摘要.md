## 本轮已定位并修复的关键问题

### 1. 章节语义被错误降级成伪 quotedText 锚点
- 典型 case：
  - `1.1 总体复苏态势中的数据太久了，换成2026年的`
- 原问题：
  - LLM 输出 `BLOCK + BY_QUOTED_TEXT`
  - `quotedText=总体复苏态势中的数据`
- 处理：
  - prompt 规则收紧
  - resolver 本地语义校验收紧
  - 本地章节规范化逻辑补强

### 2. 章节编号被错误当成顶层 ordinal
- 典型 case：
  - `在1.3 客源市场结构 后插入1.4 游客偏好分析`
  - `删1.3 客源市场结构`
- 原问题：
  - 被解析成 `BY_STRUCTURAL_ORDINAL ordinal=3 scope=TOP_LEVEL_SECTION`
  - 但实际文档 `topLevelSequence` 不是“1.1/1.2/1.3”这一层，导致 unresolved
- 处理：
  - 对 section insert / delete / rewrite / move 等 section 级动作，
    优先将 `1.3 客源市场结构` 规范化为 `SECTION + BY_HEADING_TITLE`

### 3. “后插入”语义方向错误
- 典型 case：
  - `在1.3 客源市场结构 后插入1.4 游客偏好分析`
- 原问题：
  - 模型给出 `INSERT_SECTION_BEFORE_SECTION`
- 处理：
  - 对“前/后插入同级章节”加本地规范化：
    - `后插入` -> `INSERT_BLOCK_AFTER_ANCHOR`
    - `前插入` -> `INSERT_SECTION_BEFORE_SECTION`
  - 该规则已收窄，只对“明确插入同级章节”的请求生效，不影响“在1.1中新增一段文字”这种普通段落插入。

### 4. 插入成功但后端 verifier 误判失败
- 日志 case：
  - 飞书已正常插入
  - 后端报错：
    - `目标状态校验失败：插入后未能在文档中找到新增内容`
- 原因：
  - verifier 以前拿整段 Markdown 原样去 `fullText(after).contains(newContent)`，过于脆弱
- 处理：
  - 先抽新增 heading 校验
  - 再做正文片段校验
  - 最后用 block 数量增长兜底
