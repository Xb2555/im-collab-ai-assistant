#### **赛道定位与能力考察重点**

本赛道重点考察选手对 **AI Agent** 的深刻理解与驾驭能力，以及在**多端复杂环境**下的工程落地能力。我们鼓励具备以下能力与背景的选手参赛：

- 是 **AI Native** 的思考者，能将 Agent 思维融入产品设计，以 AI 驱动核心流程，而非简单堆砌 GUI。
- 具备扎实的**客户端基础**，能应对移动端与桌面端差异，保证多端体验的一致性与流畅性。
- 拥有卓越的**工程与产品能力**，能端到端交付高质量、高可用的复杂应用。

#### **具体命题：Agent-Pilot · 从 IM 对话到演示稿的一键智能闭环**

##### **背景与痛点**

在快节奏的团队协作中，一个需求往往始于一次 IM 对话（群聊/单聊均可），历经文档撰写、多方讨论、方案修改，最终沉淀为一份正式的演示文稿。这个过程充满了大量重复、繁琐的手动操作与跨应用切换，极大地消耗了团队的创造力。如果有一个智能助手，能理解对话意图，将这些步骤拆解为若干可重用的场景模块并按需编排自动串联起整个工作流，那将是生产力的巨大飞跃。

##### **任务目标与范围**

你需要设计并实现一个以 **AI Agent 为核心**的多端协同办公助手，它能驱动并串联 **IM（即时通讯）、文档、演示文稿/自由画布** 这三个核心办公套件，实现从需求捕捉到成果汇报的全链路自动化。

**核心理念**：**AI Agent 是主驾驶（Pilot），GUI 界面是仪表盘与辅助操作台（Co-pilot）**。用户通过自然语言（语音或文本）下达指令，Agent 负责理解、拆解任务并驱动各端应用完成核心操作，GUI 则用于展示进度、提供必要确认或进行精细调整。

- **必须完成项 (Must-have)**：
  - **多端协同框架**：构建一套能在**移动端（iOS 或 Android）**和**桌面端（macOS 或 Windows）**之间实时同步状态与数据的框架。一端的操作能无缝体现在另一端。
  - **Agent 驱动的模块化场景与可组合编排**：
  - 场景 A：**意图/指令入口**（IM 群聊/单聊均可，支持文本/语音），负责捕捉用户自然语言意图并启动任务。
  - 场景 B：**任务理解与规划**（LLM/Planner，不限实现），将用户意图拆解为可执行的子任务与步骤。
  - 场景 C：**文档/白板生成与编辑**，围绕需求自动生成并迭代核心文档或白板内容。
  - 场景 D：**演示稿（PPT/自由画布）生成与排练**，将已沉淀内容结构化为演示材料，并支持演练与修改。
  - 场景 E：**多端协作与一致性**（移动端+桌面端），保证跨端状态与内容的一致和冲突解决。
  - 场景 F：**总结与交付（汇报、归档）**，输出面向汇报/归档的成果（如分享链接、导出文件等）。
  - 各场景可独立成立并单独演示，也可按需求自由组合编排。参赛作品需同时满足“**AI Agent 为主、GUI 为辅**”和“**覆盖 IM、Doc，并包含 PPT 或自由画布至少一种**”的要求，不限定固定触发顺序与链路形态。
  - **自然语言交互**：支持**文本或语音**指令，覆盖任务启动、进度查询、内容生成等关键节点。
- **可选加分项 (Good-to-have)**：
  - **离线支持**：在网络断开时，用户仍可在本地进行编辑，待网络恢复后数据能智能、无冲突地合并同步。
  - **高级 Agent 能力**：Agent 能主动发起任务澄清、总结讨论、或根据上下文推荐下一步行动。
  - **富媒体与画布操作**：支持在文档/画布中通过指令插入和处理图片、表格等，或进行布局调整。
  - **第三方平台集成**：基于**飞书**或同类开放平台的 API 实现，展示对现有生态的利用与拓展能力。若选择从零开始，则考察更底层的架构设计能力。

##### **演示与验收要求**

你的最终演示必须清晰地展示以下场景。允许单场景独立演示，须至少展示一次多场景组合编排；IM 入口的群聊/单聊均可：

- **多端协同**：在移动端与桌面端之间流畅切换，展示数据与状态的**实时、双向同步**。
- **Agent 驱动主流程**：演示选手如何通过 IM 入口（群聊/单聊均可）以自然语言触发任务，由 Agent 驱动意图理解、文档/白板生成、演示稿生成等多个场景模块的编排执行，而非依赖单一固定链路。
- **Office 套件覆盖**：演示中必须明确体现对 **IM、文档、演示文稿/自由画布** 三个组件的串联与操作。
