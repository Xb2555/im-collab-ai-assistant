# 移动端登录对接文档

## 对接概览

移动端登录流程：获取授权链接 → 跳转飞书授权 → 授权后回调 App → 换取 Token

```
移动端 App                              后端                              飞书
   │                                     │                                │
   │──── GET /auth/login-url ──────────>│                                │
   │      ?client=desktop                │                                │
   │<──── {authorizationUri, state} ─────│                                │
   │                                     │                                │
   │     跳转飞书授权页面                  │                                │
   └────────────────────────────────────────────────────────────────> │
   │                                     │                                │
   │     飞书授权后跳转回 App              │                                │
   │     agentpilot://callback           │                                │
   │     ?code=xxx&state=xxx             │                                │
   │                                     │                                │
   │──── POST /auth/callback ───────────>│                                │
   │      {code, state}                  │                                │
   │<──── {token, tokenType, expiresIn, ──│                                │
   │       user: {openId, name, avatar}} │                                │
   │                                     │                                │
```

## API 接口

### 1. 获取授权链接

**请求**
```
GET /auth/login-url?client=desktop
```

| 参数 | 必填 | 说明 |
|------|------|------|
| client | 否 | 固定填 `desktop`，不填默认 web |

**响应**
```json
{
  "code": 0,
  "data": {
    "authorizationUri": "https://open.feishu.cn/...",
    "state": "随机字符串"
  }
}
```

移动端需要将 `authorizationUri` 拼接到飞书授权 URL，跳转过去让用户授权。

### 2. 授权回调

飞书授权成功后，会回调以下地址并带 code 和 state：

```
agentpilot://callback?code=xxx&state=xxx
```

移动端 App 需要拦截这个 Deep Link，解析出 code 和 state，然后调用第 3 步。

### 3. 换取 Token

**请求**
```
POST /auth/callback
Content-Type: application/json

{
  "code": "飞书返回的code",
  "state": "之前获取的state"
}
```

**响应**
```json
{
  "code": 0,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "user": {
      "openId": "ou_xxxxx",
      "name": "张三",
      "avatarUrl": "https://xxx.com/avatar.png"
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| token | JWT 业务 Token，后续请求需要放在 Header |
| tokenType | 固定 `Bearer` |
| expiresIn | Token 有效期（秒），默认 2 小时 |
| user | 当前登录用户信息 |

### 4. 获取当前用户

**请求**
```
GET /auth/me
Authorization: Bearer <token>
```

**响应**
```json
{
  "code": 0,
  "data": {
    "openId": "ou_xxxxx",
    "name": "张三",
    "avatarUrl": "https://xxx.com/avatar.png"
  }
}
```

### 5. 登出

**请求**
```
POST /auth/logout
Authorization: Bearer <token>
```

**响应**
```json
{
  "code": 0,
  "data": true
}
```

## 对接示例

### iOS (Swift)

```swift
// 1. 获取授权链接
func getLoginUrl() async throws -> (url: String, state: String) {
    let url = URL(string: "\(baseURL)/auth/login-url?client=desktop")!
    let (data, _) = try await URLSession.shared.data(from: url)
    let response = try JSONDecoder().decode(LoginUrlResponse.self, from: data)
    return (response.data.authorizationUri, response.data.state)
}

// 2. 跳转飞书授权
func openFeishuAuth(_ url: String) {
    UIApplication.shared.open(URL(string: url)!)
}

// 3. 拦截 Deep Link 并换取 Token
func handleCallback(code: String, state: String) async throws -> TokenResponse {
    var request = URLRequest(url: URL(string: "\(baseURL)/auth/callback")!)
    request.httpMethod = "POST"
    request.setValue("application/json", forHTTPHeaderField: "Content-Type")
    request.httpBody = try JSONEncoder().encode(["code": code, "state": state])

    let (data, _) = try await URLSession.shared.data(for: request)
    return try JSONDecoder().decode(TokenResponse.self, from: data)
}
```

### Android (Kotlin)

```kotlin
// 1. 获取授权链接
suspend fun getLoginUrl(): LoginUrlResponse {
    return client.get("$BASE_URL/auth/login-url?client=desktop")
        .body()
}

// 2. 跳转飞书授权
fun openFeishonAuth(url: String) {
    val intent = Intent(Intent.ACTION_VIEW, Uri.parse(url))
    startActivity(intent)
}

// 3. 拦截 Deep Link 并换取 Token
suspend fun exchangeToken(code: String, state: String): TokenResponse {
    return client.post("$BASE_URL/auth/callback") {
        contentType(ContentType.Application.Json)
        setBody(mapOf("code" to code, "state" to state))
    }.body()
}
```

## 常见问题

### Q1: 移动端使用什么 redirect_uri？
移动端统一使用 `desktop` 回调，配置了 `agentpilot://callback` Deep Link。

### Q2: token 过期了怎么办？
当前未提供刷新 Token 接口，token 过期后需要用户重新登录。

### Q3: 报错 "Invalid or expired oauth state"？
state 是一次性的，校验后会删除，不能重复使用。确保登录和回调使用的是同一个 state。

### Q4: 报错 "redirect_uri mismatch"？
检查开发者后台的 redirect_uri 配置是否包含移动端对应的回调地址。
